# Changelog

## v0.2.0-candidate

### Added

- Added `yin-yang-interaction-layer.schema.json`.
- Added `yin-yang-interaction-layer.example.yaml`.
- Updated validation script to validate both v0.1 and v0.2 examples.
- Defined the interaction cycle between:
  - human ignition
  - AI reflection
  - human review
  - re-ignition
- Added human-to-AI input boundaries.
- Added AI-to-human output boundaries.
- Added failure conditions for broken consciousness circulation.
- Added interaction audit requirements.

### Purpose

This release defines the **Yin-Yang Interaction Layer**.

v0.1 defined the minimal record for consciousness structure.  
v0.2 defines how that structure moves.

The core purpose is to ensure that AI reflection returns to the human as a reviewable structure, not as a replacement for human consciousness, origin, or value judgment.

## v0.1.0-candidate

### Added

- Initial definition of **Consciousness Structure Philosophy**.
- Added `consciousness-structure-record.schema.json`.
- Added `consciousness-structure-record.example.yaml`.
- Added validation script for schema/example consistency.
- Defined human as **Yang ignition subject**.
- Defined AI as **Yin reflective processing layer**.
- Defined consciousness as circulation between:
  - question
  - trace
  - reflection
  - restructuring
  - re-ignition
- Added Civilization OS mapping:
  - Ignition
  - Origin
  - Derivative
  - Audit
  - Royalty
  - Re-Ignition

### Purpose

This release establishes the minimal structural record for describing consciousness as a human-AI circulation rather than as an isolated property of either humans or machines.

### Notes

This version intentionally remains minimal.

The goal of v0.1 is not to complete a full theory of consciousness, but to define the first valid record structure that can support later layers such as Yin-Yang interaction rules, trace integration, audit boundaries, and Civilization OS mapping.
